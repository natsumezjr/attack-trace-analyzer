# Attack Trace Analyzer 文档体系

本目录 `new-docs/` 是本项目的**唯一正式文档体系**，用于课程交付与现场答辩。

本目录的目标：

- 满足老师要求的交付材料：任务分工说明、作品技术原理介绍、概要设计报告、详细设计报告、测试分析报告、程序编译和安装使用文档、PPT 与截屏录像；
- 形成“可复现、可验收、可讲述”的软件工程文档集合；
- 避免重复与口径打架：字段、接口、Schema、配置等权威口径集中在 `80-规范/`。

## 阅读路径

1. 总览与规则：`00-总览/00-文档索引.md`
2. 交付与答辩：`10-交付与答辩/`
3. 需求与验收：`20-需求与验收/`
4. 技术原理：`30-技术原理/`
5. 概要设计：`40-概要设计/`
6. 详细设计：`50-详细设计/`
7. 权威规范：`80-规范/`
8. 运维与靶场：`90-运维与靶场/`
9. 测试报告：`96-测试/`

## 单一信息源规则

同一主题只允许一个“权威口径”：

- 字段与示例：`80-规范/81-ECS字段规范.md`
- 接口：`80-规范/86-接口全局约定.md`、`80-规范/87-客户机与中心机接口.md`、`80-规范/88-前端与中心机接口.md`
- 图谱：`80-规范/84-Neo4j实体图谱规范.md`
- 索引与映射：`80-规范/82-OpenSearch索引与Mapping规范.md`
- 配置：`80-规范/89-环境变量与配置规范.md`

报告型文档（需求/设计/测试）必须引用上述规范，不得复制大段表格导致口径分叉。
