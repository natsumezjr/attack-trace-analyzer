# 靶场指南对齐说明

本文件用于说明 `guide_docs/` 与 `new-docs/90-运维与靶场/` 的关系与口径对齐规则，避免同一主题出现两套互相矛盾的描述。

## 1. 权威入口（固定）

本仓库对外的靶场运维权威入口固定为 `new-docs/90-运维与靶场/`，其目录内文件用于课程交付与现场演示：

1. `90-编译安装与使用.md`
2. `91-靶场部署.md`
3. `92-一键编排.md`
4. `93-C2部署与证据点.md`
5. `94-验证清单.md`
6. `95-重置复现与排障.md`
7. `96-靶场交付说明.md`
8. `97-单机靶场落地步骤.md`

## 2. guide-docs 与 new-docs 对齐规则（固定）

1. `guide_docs/` 用于靶场负责人交付与过程记录；`new-docs/` 用于课程交付材料与最终口径。
2. 两套文档对同一事实（IP、端口、拓扑、命令、目录）必须保持一致。
3. SSH 登录凭据直接保存在 `96-靶场交付说明.md` 和 `98-WSL连接服务器指南.md` 中。

## 3. 文件映射（固定）

`guide_docs/` 目录内文件名包含括号与中英文混排，为避免在 `new-docs/` 中引入不确定措辞，本节使用 guide-docs 的编号进行映射。

| guide-docs 编号 | new-docs 文件 | 说明 |
|---:|---|---|
| 00 | `90-编译安装与使用.md` | 指南入口统一到 new-docs |
| 01 | `91-靶场部署.md` | 节点、端口、目录统一 |
| 02 | `92-一键编排.md` | 启停顺序与命令统一 |
| 03 | `93-C2部署与证据点.md` | C2 的网络与证据点统一 |
| 04 | `94-验证清单.md` | 验收命令与通过标准统一 |
| 05 | `95-重置复现与排障.md` | 演示前重置流程统一 |
| 10 | `97-单机靶场落地步骤.md` | 单机落地步骤固化为 new-docs |
| 20 | `96-靶场交付说明.md` | 交付口径与端口表统一；不保存明文口令 |
| 21 | `92-一键编排.md`、`94-验证清单.md`、`95-重置复现与排障.md` | 运行手册拆分到启停、验收、重置 |
