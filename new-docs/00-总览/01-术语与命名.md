# 术语与命名

本文件用于统一项目术语与命名，避免在需求、设计、实现、演示与测试中出现歧义。

## 1. 术语表

| 术语 | 英文 | 定义 |
|---|---|---|
| 客户机 | Client | 部署在靶场各节点上的采集与缓冲组件，负责采集 Falco、Filebeat、Suricata 等数据并对外提供拉取接口。 |
| 中心机 | Center | 部署在靶场中心节点上的服务端系统，负责轮询拉取客户机数据、写入存储、检测融合、入图、对外 API、溯源任务执行与报告导出。 |
| Telemetry | Telemetry | 事实事件层，描述“发生了什么”，不做善恶判断；在 ECS 中满足 `event.kind="event"`。 |
| Raw Finding | Raw Finding | 原始告警层，来源于传感器规则或检测引擎；在 ECS 中满足 `event.kind="alert"` 且不属于 Canonical。 |
| Canonical Finding | Canonical Finding | 规范告警层，由中心机对 Raw Findings 融合去重生成；在 ECS 中满足 `event.kind="alert"` 且 `event.dataset="finding.canonical"`。 |
| 实体图谱 | Entity Graph | Neo4j 中的实体关系图，节点与边的类型、唯一键与属性规范由图谱规范定义。 |
| 溯源任务 | Trace Task | 前端点选节点触发的异步任务，读取子图并执行溯源算法，结果写回 Neo4j 边属性并更新任务状态。 |
| 证据链 | Evidence | 支撑溯源结论的证据集合，最小可回溯单位为 ECS 事件的 `event.id`。 |

## 2. 命名约定

### 2.1 文件与目录命名

- 仅使用数字、英文字母、汉字与连字符 `-`
- 目录使用“分类名”，文件使用“编号-主题”
- 同一层级下按编号排序

### 2.2 核心标识命名

| 名称 | 约定 |
|---|---|
| `event.id` | 全局幂等键，必须稳定，可用于证据回溯与去重；生成规则见 `80-规范/81-ECS字段规范.md`。 |
| `host.id` | 节点唯一标识，必须跨数据源一致；生成与来源见 `80-规范/81-ECS字段规范.md`。 |
| `process.entity_id` | 进程唯一标识，用于跨日志/行为把同一进程串联；见 `80-规范/81-ECS字段规范.md`。 |
| `task.id` | 溯源任务唯一标识，格式见 `50-详细设计/分析/70-任务模型与状态机.md`。 |

### 2.3 索引与图谱命名

- OpenSearch 索引命名规则见 `80-规范/82-OpenSearch索引与Mapping规范.md`
- Neo4j 节点 label、关系类型、唯一约束见 `80-规范/84-Neo4j实体图谱规范.md`
