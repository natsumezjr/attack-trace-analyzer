# 文档索引

本目录为本项目的**唯一正式文档体系**。所有课程交付材料中的“文档报告”和“安装使用说明”均以 `new-docs/` 为准。

## 1. 文档分层与权威口径

为避免内容重复与口径打架，本项目文档按以下分层组织：

1. **交付与答辩**：对齐课程要求与评分口径，给出提交包结构、PPT 大纲、演示与录屏脚本。
2. **报告与设计**：需求分析、概要设计、详细设计、测试分析等报告类文档，强调可验收与可讲述。
3. **规范**：字段、接口、索引、图谱、配置的唯一权威口径，任何逐字段表格只允许出现在 `80-规范/`。
4. **运维与靶场**：编译安装、靶场部署、一键编排、验证清单、复现与排障等可执行操作文档。

## 2. 推荐阅读路径

按答辩准备顺序阅读：

1. 交付与答辩：`10-交付与答辩/`
2. 需求与验收：`20-需求与验收/`
3. 技术原理：`30-技术原理/`
4. 概要设计：`40-概要设计/`
5. 详细设计：`50-详细设计/`
6. 权威规范：`80-规范/`
7. 运维与靶场：`90-运维与靶场/`
8. 测试报告：`96-测试/`

## 4. 引用规则

### 4.1 单一信息源

同一主题只允许一个权威文档：

- ECS 字段：`80-规范/81-ECS字段规范.md`
- 接口：`80-规范/86-接口全局约定.md`、`80-规范/87-客户机与中心机接口.md`、`80-规范/88-前端与中心机接口.md`
- OpenSearch：`80-规范/82-OpenSearch索引与Mapping规范.md`
- Neo4j 图谱：`80-规范/84-Neo4j实体图谱规范.md`
- 配置：`80-规范/89-环境变量与配置规范.md`

报告与设计文档引用规范时，只允许：

- 引用该规范的章节标题；
- 摘要性说明其对本章结论的约束；
- 在必要时给出最小示例（示例必须与规范一致）。

### 4.2 冲突处理

迁移与写作过程中发现的任何矛盾（文档↔文档、文档↔代码、代码↔运行行为）必须记录到：

- `00-总览/02-冲突与决议.md`

正文不允许同时保留两种口径，不允许用“说明：另一处不同”这种方式打补丁。
