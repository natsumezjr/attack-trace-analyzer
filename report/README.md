# Report 目录说明

本目录用于存放课程交付材料的**最终版本**和**PDF 打包文件**。

## 📁 目录结构

```
report/
├── README.md                        # 本文件
├── 2025课程设计.pdf                 # 课程设计要求文档
├── 课程设计要求文档.pdf             # 课程设计要求文档（备份）
├── 作品技术原理介绍.pdf             # 技术原理介绍 PDF
├── 任务分工说明--待完善.docx        # 任务分工说明 Word 版本
└── ppt_visual_design.json           # PPT 视觉设计配置
```

## 📝 与 docs/ 的区别

### `docs/` 目录
- **用途**：项目开发过程中的**所有文档**（Markdown 格式）
- **特点**：版本控制、便于修改、实时更新
- **内容**：
  - 需求分析、设计文档、技术原理
  - 规范说明、运维文档、测试文档
  - 交付与答辩相关文档

### `report/` 目录
- **用途**：课程**最终交付材料**（PDF 和其他格式）
- **特点**：正式版本、打包交付、不可修改
- **内容**：
  - PDF 版本的设计文档
  - 课程要求的正式表格
  - PPT 设计配置

## 📦 交付包打包

当需要打包交付材料时，本目录下的所有文件应包含在交付包中。

### 交付清单

1. **设计文档**：
   - 从 `docs/` 导出的 PDF 版本

2. **课程要求表格**：
   - 任务分工说明（`docs/10-交付与答辩/12-任务分工说明.md`）
   - 课程设计要求文档

3. **演示材料**：
   - PPT 文件
   - 录像文件

## 🔗 相关文档

- **所有 Markdown 文档**：`../docs/`
- **交付与答辩**：`../docs/10-交付与答辩/`
- **技术原理**：`../docs/30-技术原理/30-作品技术原理介绍.md`

## ⚠️ 注意事项

1. **不要在 `report/` 中存放 Markdown 源文件**
   - Markdown 文档应放在 `docs/` 对应位置
   - 本目录只存放 PDF 和正式交付材料

2. **保持同步**
   - 当 `docs/` 中的文档更新后，及时导出新的 PDF 到本目录
   - 确保交付材料是最新版本

3. **版本管理**
   - PDF 文件不建议提交到 Git（已在 `.gitignore` 中配置）
   - 仅在交付时打包生成
