# 文档模板

本文件提供本项目文档的统一模板，保证结构一致、引用一致、口径一致。

## 1. 通用头部模板

所有文档都必须包含以下信息：

- 文档目的：本文件解决什么问题
- 读者对象：老师答辩、项目成员、靶场负责人、测试负责人等
- 引用关系：本文件依赖哪些权威规范，哪些内容不在本文件重复
- 生效范围：本文件适用的代码目录与运行形态（client、backend、frontend、靶场）

头部结构固定为如下：

```
# 标题

## 文档目的

## 读者对象

## 引用关系

## 生效范围
```

## 2. 报告类模板

适用：需求分析报告、概要设计报告、详细设计报告、测试分析报告、技术原理介绍。

报告类文档强调“叙事完整 + 可验收”，不承载逐字段表格。

```
## 文档目的

## 读者对象

## 引用关系
- 字段规范：80-规范/81-ECS字段规范.md
- 接口规范：80-规范/86-接口全局约定.md 等

## 范围与边界

## 关键设计结论

## 核心流程与机制

## 可验收性
- 关联验收用例：20-需求与验收/21-验收标准与验收用例.md
- 关联演示脚本：10-交付与答辩/13-PPT大纲与演示脚本.md

## 风险与限制
```

## 3. 规范类模板

适用：ECS 字段规范、索引与 mapping 规范、图谱规范、接口规范、配置规范。

规范类文档是“单一信息源”，必须写清楚固定规则、失败处理与示例。

```
## 文档目的

## 权威性声明
本文件是本主题唯一权威口径。

## 适用范围

## 定义与约束

## 数据结构或协议

## 生成规则

## 校验与失败处理

## 示例

## 版本与变更记录
```

## 4. 操作类模板

适用：编译安装与使用、靶场部署、一键编排、验证清单、重置复现与排障。

操作类文档必须写到“命令级别”，并给出每一步的预期现象与失败判定。

```
## 目标与交付标准

## 环境前置

## 操作步骤

## 预期结果与验证方法

## 失败判定与修复动作

## 证据与截图清单
```
