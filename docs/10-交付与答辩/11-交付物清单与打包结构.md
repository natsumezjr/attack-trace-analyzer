# 交付物清单与打包结构

## 文档目的

本文件定义本项目最终提交包的目录结构、文件命名规则与自检清单，确保交付物齐全、结构清晰、可被快速检查评分。

## 读者对象

- 组长与负责打包提交的同学
- 负责答辩与展示的同学

## 引用关系

- 课程要求原文与时间地点：`10-课程设计要求与评分口径.md`
- 各报告正文：见本目录 `docs/` 对应章节

## 提交包命名

提交包文件名固定为：

- `组长班级-组长姓名-学号.zip`

## 提交包目录结构

提交包内目录结构固定为：

```
01-任务分工说明
02-作品技术原理介绍
03-概要设计报告
04-详细设计报告
05-测试分析报告
06-程序编译安装与使用文档
07-程序源代码
08-PPT
09-截屏录像
```

## 交付物与仓库路径映射

| 提交包目录 | 内容来源 | 仓库路径 |
|---|---|---|
| 01-任务分工说明 | PDF 或 Markdown 导出 | `docs/10-交付与答辩/12-任务分工说明.md` |
| 02-作品技术原理介绍 | 报告正文 | `docs/30-技术原理/30-作品技术原理介绍.md` |
| 03-概要设计报告 | 报告正文 | `docs/40-概要设计/` |
| 04-详细设计报告 | 报告正文 | `docs/50-详细设计/` |
| 05-测试分析报告 | 报告正文 | `docs/96-测试/` |
| 06-程序编译安装与使用文档 | 报告正文 | `docs/90-运维与靶场/90-编译安装与使用.md` |
| 07-程序源代码 | 源码 | 仓库根目录（`backend/`、`client/`、`frontend/`） |

| 09-截屏录像 | 视频与截图 | `docs/10-交付与答辩/14-录像与截图清单.md` 对应产出 |

## 最终自检清单

打包前逐项自检，任何一项缺失都不允许提交：

1. 任务分工说明已填写完整：姓名、学号、分工与工作量占比齐全
2. 技术原理介绍可讲述：从采集到溯源的链路完整
3. 概要设计报告完整：架构、数据流、部署拓扑、非功能设计齐全
4. 详细设计报告按模块拆分完整：客户机、中心机、分析、前端均有对应文档
5. 测试分析报告包含结果：至少覆盖核心闭环与回退路径
6. 编译安装与使用文档可复现：按文档操作可以从零跑到可演示
7. PPT 与演示脚本一致：每页内容与现场操作对应
8. 录像与截图清单完成：证据点齐全且命名规范
