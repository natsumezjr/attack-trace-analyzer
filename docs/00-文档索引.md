# 文档索引（建议从此阅读）

本目录用于存放课程设计的“可交付文档”。文档之间有依赖关系，建议按顺序阅读。

## 0. 课程要求（外部约束）

- `docs/课程设计要求文档.md`  
  题目要求、组队方法、考核方式、时间地点等（本项目必须满足的外部约束）。

## 1. 需求与设计（内部实现）

- `docs/04-模块与数据流说明.md`  
  面向组员快速上手：从端侧采集到最终串链的“全流程故事线”（弱化字段细节，强调模块与技术）。

- `docs/99-选型决策记录.md`  
  把关键口径一次性“写死”（已拍板），后续实现与写报告以此为准。

- `docs/01-整体需求文档.md`  
  项目目标、范围、角色、功能需求/非功能需求、验收标准。

- `docs/02-系统规格说明书.md`  
  数据流、模块划分、输入/输出、检测与关联算法、接口与输出物（可直接用于“概要/详细设计报告”）。

- `docs/03-数据库设计.md`  
  03 系列总览：数据对象、流转路径、入库路由规则（ECS + 接口 + 存储）。

- `docs/03A-ECS字段规范.md`  
  ECS 字段规范（ECS 子集 + `custom.*`）：Telemetry/Findings 的字段口径与 dataset 约定（用于统一归一化与接口对齐）。

- `docs/03B-存储与图谱设计.md`  
  存储与图谱设计：OpenSearch 索引划分/保留策略 + Neo4j 图模型 +（可选）关系型表结构/中心机元数据。

- `docs/03C-客户端中心机接口规范.md`  
  客户端 ↔ 中心机接口：客户端启动注册 + 中心机轮询拉取数据（Telemetry/Findings）。

## 2. 调研与借鉴（加速开发）

- `docs/05-开源项目调研与借鉴清单.md`  
  结合题目要求，把可复用的开源平台/组件按“能解决什么需求”“集成成本”“我们计划如何使用”进行整理。

---

## 说明

- 本项目默认以 **ECS（Elastic Common Schema）思想**做统一范式：只保留“能关联/溯源”的最小字段集合。
- 不追求“严格归因某个 APT 组织”，更合理的输出是：**基于 TTP（Technique/Tactic/Procedure）进行相似性匹配的候选列表**。
