## 重要说明

### 设计原则

- 本项目默认以 **ECS（Elastic Common Schema）思想**做统一范式：只保留"能关联/溯源"的最小字段集合
- 不追求"严格归因某个 APT 组织"，更合理的输出是：**基于离线 MITRE ATT&CK Enterprise CTI（STIX 2.1｜attack-stix-data，Intrusion Set ↔ Technique）做 TTP 相似性匹配，输出候选列表与解释性证据**

### 实现口径

- 项目口径与参数已写入各规格文档，后续以具体章节为准
- 接口以 `06C-客户端中心机接口规范.md` 为主
- 字段以 `06A-ECS字段规范.md` 为主

### 文档与代码冲突时

**以 docs/ 下的规格文档为准**。代码实现应严格遵循文档规范。
