## 1. 总体架构

### 1.1 应用入口与生命周期

**文件位置：** `backend/main.py`

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from app.api.router import api_router
from app.core.config import settings
from app.core.logging import configure_logging
from app.services.analyze.runner import start_task_runner, stop_task_runner
from app.services.client_poller import start_polling, stop_polling

configure_logging(settings.log_level)

@asynccontextmanager
async def lifespan(_: FastAPI):
    # 应用启动时开启后台任务
    await start_polling()
    await start_task_runner()
    try:
        yield
    finally:
        # 应用关闭时停止后台任务
        await stop_task_runner()
        await stop_polling()

app = FastAPI(title=settings.app_name, version=settings.app_version, lifespan=lifespan)
app.include_router(api_router)
```

**生命周期保证：**

1. 应用启动时自动启动中心机轮询任务（`start_polling()`）
2. 应用启动时自动启动 Analysis 异步任务 runner（`start_task_runner()`）
3. 应用关闭时优雅停止上述后台任务（`stop_*()`），不阻塞 HTTP 请求处理

## 3. 客户机注册表与轮询服务

中心机轮询服务不维护“内存注册表”，而是以 OpenSearch `client-registry` 作为**权威注册表**：

- `POST /api/v1/clients/register`（完整版注册）与 `POST /api/v1/targets/register`（简化版注册）都会写入该索引；
- 轮询服务每个 tick 读取 `client-registry`，按客户机声明的 `listen_url/capabilities` 去拉取事件；
- 拉取到的 ECS 文档写入 OpenSearch（Telemetry/Findings），并可选触发检测+融合（Step 3）。

### 3.1 注册表（OpenSearch）

**索引名：** `client-registry`（固定索引，不按日滚动）

**字段口径：** 见 `docs/31-OpenSearch模块规格说明书.md` 与 `docs/54-客户机-中心机接口规范.md`

### 3.2 后台轮询服务

**文件位置：** `backend/app/services/client_poller.py`

**环境变量：**（权威清单见 `docs/53-环境变量规范.md`）

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `CENTER_POLL_INTERVAL_SECONDS` | `5` | 轮询间隔（秒），范围 [1, 3600] |
| `CENTER_POLL_TIMEOUT_SECONDS` | `5` | HTTP 请求超时（秒） |
| `CENTER_RUN_ANALYSIS_EACH_TICK` | `0` | 是否每个 tick 执行检测+融合（`1/true/yes` 启用） |

**轮询流程（概要）：**

```python
async def _poll_loop(stop_event: asyncio.Event) -> None:
    async with httpx.AsyncClient(timeout=_poll_timeout()) as http:
        while not stop_event.is_set():
            for doc in _list_registered_clients():
                client_id, listen_url, caps = _extract_client_info(doc)
                if not client_id or not listen_url:
                    continue
                await _poll_client(http, client_id, listen_url, caps)
            await asyncio.wait_for(stop_event.wait(), timeout=_poll_interval_seconds())
```

**单个客户机轮询（`_poll_client`）：**

1. 按 `capabilities` 选择要拉取的 routes（默认 `falco/suricata/filebeat`）  
2. 对每个 route 请求：`GET {listen_url}/{route}`（例如 `http://10.0.0.11:8888/falco`）  
3. 响应为 `{"total": N, "data": [...]}` 时，提取 `data` 并调用 `store_events()` 写入 OpenSearch  
4. 若开启 `CENTER_RUN_ANALYSIS_EACH_TICK`，则调用 `run_data_analysis()` 执行检测+融合  
5. 写回 `poll.*` 字段（`poll.last_seen/status/last_error`）用于观察轮询状态
